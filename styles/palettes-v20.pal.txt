>##  PALETTE DEFINITION
>##    Each new palette starts with a line prefixed by ">>>" followed by
>##    the palette name. Example: ">>> Ocean Breeze"
>##
>##  COLOR ENTRY FORMAT
>##    Every line under a palette name represents a single color.
>##    Fields must be separated by colons (:) using the following structure:
>##
>##       <HEX_CODE> : <COLOR_NAME> : <OPTIONAL_TEXTURE> : <OPTIONAL_OBJECT>
>##
>##    * HEX_CODE         : Hexadecimal color code (e.g., #00FFFF). Mandatory.
>##    * COLOR_NAME       : Simple english name for the color. Mandatory.
>##    * OPTIONAL_TEXTURE : Descriptive texture string. Can be left empty.
>##    * OPTIONAL_OBJECT  : Descriptive object/concept string. Can be left empty.
>##
>##  PALETTE STRUCTURE & GRADIENT RULES
>##    Color Count : A standard palette MUST contain exactly 4 colors.
>##    Color Order : Order colors sequentially from HIGHEST illumination (brightest)
>##                  to LOWEST illumination (darkest).
>##    5th Color   : An optional 5th color can be added at the end of the palette:
>##                   - It does NOT follow the gradient rule (any intensity allowed).
>##                   - It serves as a "Special Highlight Color".
>##                   - Usage: Linework, accents or any special effect
>##                   - This 5th element will be used in specific cases.
>##
